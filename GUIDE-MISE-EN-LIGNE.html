🚀 Guide de mise en ligne

SOS Informatique 44 — Comment mettre votre site en ligne sur GitHub Pages + IONOS
0
Rendre le depot public
⏱ 2 min
🌐
Ouvrez votre navigateur et allez sur :
https://github.com/patatou97/sosinformatique44
⚙
En haut, cliquez sur l'onglet Settings (l'engrenage)
👇
Descendez tout en bas de la page jusqu'a la zone rouge "Danger Zone"
🔄
Cliquez sur Change repository visibility
🔓
Selectionnez Public
⌨
GitHub vous demande de taper le nom pour confirmer. Tapez :
patatou97/sosinformatique44
✅
Cliquez sur I understand, change repository visibility
✅
Pas de souci de securite ! Le code d'un site web est toujours visible par les navigateurs. Votre cle API n'est PAS dans le code, elle est dans votre navigateur uniquement.
1
Activer GitHub Pages
⏱ 5 min
⚙
Vous etes toujours dans Settings. Dans le menu a gauche, cliquez sur Pages
📦
Sous "Build and deployment", dans Source :
Selectionnez Deploy from a branch
🌳
Sous Branch :
• Cliquez sur le menu qui dit "None"
• Selectionnez main
• Laissez le dossier sur / (root)
💾
Cliquez sur Save
⏳
Attendez 2-3 minutes. Rechargez la page (touche F5).
🎉
Un message vert apparait :
✔ Your site is live at https://patatou97.github.io/sosinformatique44/
Cliquez dessus pour verifier que votre site s'affiche !
🏆
Bravo ! Votre site est deja en ligne a cette adresse. Mais on veut utiliser votre vrai nom de domaine. Passez a l'etape 2.
2
Dire a GitHub d'utiliser votre domaine
⏱ 2 min
📝
Toujours dans Settings > Pages, descendez jusqu'a "Custom domain"
⌨
Dans le champ, tapez :
sos-informatique44.fr
💾
Cliquez Save
⚠
Un message rouge va apparaitre du style "DNS check unsuccessful". C'est NORMAL ! On n'a pas encore configure IONOS. Laissez cette page ouverte et passez a l'etape 3.
3
Configurer IONOS (le DNS)
⏱ 10 min
🌐
Dans un nouvel onglet, allez sur :
https://my.ionos.fr
Connectez-vous avec vos identifiants IONOS
📁
Cliquez sur Domaines & SSL dans le menu
🔍
Cliquez sur sos-informatique44.fr
🔧
Cliquez sur DNS (ou "Gerer les enregistrements DNS")
⚠
ATTENTION : Vous allez voir une liste d'enregistrements existants. Ne touchez PAS aux lignes MX (email) ou TXT. On modifie UNIQUEMENT les lignes A et CNAME.

❌ D'abord : Supprimer l'ancien enregistrement A

🗑
Si vous voyez une ligne avec :
• Type : A
• Nom : @
• Valeur : une IP d'IONOS (genre 217.xx.xx.xx)

Supprimez-la (icone poubelle a droite)

➕ Ensuite : Ajouter les 4 enregistrements A de GitHub

➕
Cliquez Ajouter un enregistrement et creez ces 4 lignes une par une :
TypeNom / HostValeur / Pointe vers
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
🔁
Pour chaque ligne :
• Type = A
• Nom = @ (ou laissez vide)
• Valeur = l'adresse IP du tableau
• TTL = 1 heure (ou par defaut)
• Cliquez Enregistrer

Repetez 4 fois, une ligne pour chaque IP.

🔗 Enfin : Ajouter le CNAME pour www

🔗
Cherchez la ligne CNAME avec Nom www.
Si elle existe : Modifiez-la
Si elle n'existe pas : Ajoutez-la
TypeNom / HostValeur / Pointe vers
CNAMEwwwpatatou97.github.io
💾
Cliquez Enregistrer

📋 Votre DNS doit ressembler a ca :

TypeNomValeur
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwpatatou97.github.io
MX, TXT, etc....NE PAS TOUCHER
4
Attendre la propagation
⏱ 10-30 min
☕
Prenez un cafe ! Le DNS met 10 a 30 minutes a se propager sur Internet. Parfois jusqu'a 1 heure.
🔍
Pour verifier, ouvrez un nouvel onglet et tapez :
sos-informatique44.fr
✅
Si vous voyez votre site : c'est bon, passez a l'etape 5 !
Si ca ne marche pas : attendez encore 10 minutes et reessayez.
5
Activer HTTPS (le cadenas vert)
⏱ 1 min
🔙
Retournez sur GitHub : Settings > Pages
🟢
Le message rouge a disparu. A la place : un message vert.
☑
Cochez la case ☑ Enforce HTTPS
⏳
Attendez 5-10 minutes. GitHub genere un certificat SSL gratuit.
🔒
Verifiez : allez sur https://sos-informatique44.frVous devez voir un cadenas dans la barre d'adresse.
6
Verifier que tout marche
⏱ 2 min
🏠
Ouvrez : https://sos-informatique44.fr ➡ Vous devez voir votre site principal
📰
Ouvrez : https://sos-informatique44.fr/blog.html ➡ Vous devez voir la page blog publique
🔧
Ouvrez : https://sos-informatique44.fr/blog-sos-informatique44.html ➡ Vous devez voir le blog admin (CRM, Veille IA, etc.)
Cette adresse c'est VOTRE outil, ne la donnez a personne.
🎉
FELICITATIONS ! Votre site est en ligne !
Les clients peuvent maintenant vous trouver sur sos-informatique44.fr

📋 Resume en 7 etapes

0
GitHub Settings > Danger Zone > Passer en Public
1
GitHub Settings > Pages > Branch main > Save
2
GitHub Pages > Custom domain > sos-informatique44.fr > Save
3
IONOS DNS > 4 enregistrements A + 1 CNAME www
4
Attendre 10-30 minutes
5
GitHub Pages > Cocher Enforce HTTPS
6
Tester sos-informatique44.fr ✔
⭐
Bonus : Mettre a jour le site au quotidien
📝
Quand on modifie ensemble les fichiers et qu'on fait git push, le site est automatiquement mis a jour en 1-2 minutes sur GitHub Pages.
🚫
Vous n'avez RIEN a faire sur IONOS a chaque fois. La configuration DNS c'est une seule fois, c'est fait.
